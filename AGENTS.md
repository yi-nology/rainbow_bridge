# AGENTS.md

## Project Summary

Rainbow Bridge (虹桥计划) is a self-hosted configuration management platform. Go backend (CloudWeGo Hertz + GORM) serves a Vue 3 frontend and a REST API. Default DB is SQLite; MySQL and PostgreSQL are also supported.

## Repository Layout

```
main.go, initialize.go    App entrypoint and DI bootstrap
biz/                      Core Go backend
  handler/                HTTP handlers per domain module
  router/                 Route registration (generated by hz)
  service/                Business logic layer
  dal/                    Data access (db/ DAO, model/ GORM entities)
  model/                  Protobuf-generated message types (*.pb.go) — DO NOT EDIT
  middleware/             Hertz middleware (CORS, auth, logging, recovery)
idl/biz/                  Proto definitions for all API modules
vue/                      Vue 3 + Vite + Tailwind frontend
pkg/                      Shared packages (config, database, static, logging, redis, lock, etc.)
tests/e2e/                Playwright E2E tests
tests/performance/        k6 load tests
deploy/                   Docker Compose, Kubernetes, and Nginx configs
script/                   Build and utility scripts
```

## Build & Run

```bash
# Backend — production mode (static files embedded in binary)
go build -o output/bin/hertz_service .

# Backend — dev mode (serves frontend from filesystem, supports hot reload)
go build -tags=dev -o output/bin/hertz_service .

# Run
./output/bin/hertz_service --config config.yaml

# Frontend dev server (port 3000, proxies /rainbow-bridge/api to backend :8080)
cd vue && npm run dev

# Full app build (frontend + backend embedded) — uses build.sh
BASE_PATH=/rainbow-bridge BUILD_TARGET=app ./build.sh

# Dev build script
BUILD_MODE=dev ./build.sh

# Make targets
make build          # Full app (frontend + Go binary)
make build-frontend # Vue frontend only
make test           # Go tests with coverage
make lint           # golangci-lint
make docker         # Build all Docker images
make dev            # docker compose up (SQLite variant)
```

## Key Architecture Facts

- **Generated code**: All files under `biz/model/` (`*.pb.go`) and `biz/router/` are generated by `hz` from `idl/biz/*.proto`. Regenerate with `hz update -idl idl/biz/*.proto`. Never edit these directly.
- **Static file serving** uses build tags: `pkg/static/static.go` (prod, `!dev` tag, uses `embed.FS`) vs `pkg/static/static_dev.go` (dev tag, reads from filesystem). The `FRONTEND_DIR` env var overrides the dev-mode search path.
- **Build-time ldflags** inject `Version`, `GitCommit`, `BuildTime`, `BasePath` into `main.*` vars.
- **BasePath** (URL prefix like `/rainbow-bridge`) is resolved: config file `server.base_path` > env var `BASE_PATH` > build-time ldflags.
- **Service initialization** (`initialize.go`): loads config → inits logging → opens DB → auto-migrates tables → seeds defaults → optionally connects Redis → registers middleware → registers routes.
- **Handler DI**: `biz/router/register.go` calls `SetService(svc)` on each handler package. Handlers are stateful singletons receiving a `*service.Service`.
- **Frontend is Vue 3** (not React/Next.js despite README references to Next.js). Uses Vite, Pinia, Radix Vue, Tailwind CSS. Output goes to `vue/dist/`, copied to `pkg/static/web/` for embedding.
- **Vite base path** is set via `BASE_PATH` env var (defaults to `/rainbow-bridge/`).

## Testing

```bash
# Go unit tests
go test ./...
go test -v -race -coverprofile=coverage.out -covermode=atomic ./...

# E2E tests (requires running server)
cd tests/e2e
npm install && npx playwright install
npm test

# Performance tests (requires k6)
k6 run tests/performance/api-load-test.js
```

## API Quick Reference

All endpoints under `/api/v1/`:
- `GET /ping` — health check
- `GET /api/v1/version` — version info
- Environment: `/api/v1/environment/{list,create,update,delete}`
- Pipeline: `/api/v1/pipeline/{list,create,update,delete}`
- Config: `/api/v1/config/{list,create,update,delete,detail}`
- Asset: `/api/v1/asset/{list,upload,file/{file_id}}`
- Runtime: `GET /api/v1/runtime/config` (headers: `x-environment`, `x-pipeline`), `GET /api/v1/runtime/static`
- Transfer: `/api/v1/transfer/{export,export-tree,import,import-preview,import-selective,migrate}`

## Important Conventions

- Config `alias` field is immutable after creation; `name` can be changed freely.
- Asset references use `asset://{file_id}` prefix in config content; exported static packages replace these with file paths.
- SQLite requires CGO for cross-compilation; use `script/build_linux_amd64.sh` or `CGO_ENABLED=0` for pure-Go builds.
- Redis is optional but enables distributed write locks and caching when enabled in `config.yaml`.
- MinIO storage is optional; configured under `storage` in `config.yaml`.
- Docker images: `ghcr.io/yi-nology/rainbow_bridge-api`, `rainbow_bridge-app`, `rainbow_bridge-frontend`.
- Release workflow triggers on `v*` tags pushing to GitHub.
