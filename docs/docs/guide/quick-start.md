# 快速开始

欢迎使用虹桥计划！本指南将帮助你在 5 分钟内快速体验系统的核心功能。

## 🎯 前置准备

确保你已经完成了 [安装部署](/guide/installation)。如果还没有，请先完成部署。

## 🚀 5 分钟快速体验

### 步骤 1：访问管理界面（1 分钟）

打开浏览器访问：http://localhost:8080/rainbow-bridge

你将看到项目首页，包含：
- 项目简介
- 核心功能特性
- 运行时配置演示

### 步骤 2：创建第一个环境（1 分钟）

1. 点击左侧导航栏的 **环境管理**
2. 点击 "新建环境" 按钮
3. 填写环境信息：
   - **环境标识**: `dev`
   - **环境名称**: `开发环境`
   - **备注**: `本地开发测试环境`
4. 点击 "确定" 保存

✅ 恭喜！你创建了第一个环境。

### 步骤 3：创建第一个渠道（1 分钟）

1. 在环境列表中找到 `开发环境`
2. 点击该环境的 "管理渠道" 按钮
3. 点击 "新建渠道"
4. 填写渠道信息：
   - **渠道标识**: `main`
   - **渠道名称**: `主渠道`
   - **备注**: `主要开发渠道`
5. 点击 "确定" 保存

✅ 现在你有了第一个渠道！

### 步骤 4：创建第一个配置（2 分钟）

1. 点击左侧导航栏的 **配置管理**
2. 选择环境：`开发环境`，渠道：`主渠道`
3. 点击 "新建配置" 按钮
4. 填写配置信息：

**示例 1：键值对配置**
```
配置类型：键值对
名称：max_retry_count
别名：最大重试次数
配置内容：
{
  "max_retry": 3,
  "timeout": 5000
}
备注：API 请求重试配置
```

**示例 2：纯文本配置**
```
配置类型：纯文本
名称：welcome_message
别名：欢迎消息
配置内容：欢迎使用虹桥计划！
备注：首页欢迎文案
```

**示例 3：色彩标签配置**
```
配置类型：色彩标签
名称：theme_color
别名：主题色
配置内容：#1677FF
备注：应用主题色
```

5. 点击 "确定" 保存每个配置

✅ 你已经掌握了配置管理！

### 步骤 5：查看运行时配置（可选）

1. 点击 **运行时配置** 页面
2. 选择环境：`开发环境`，渠道：`主渠道`
3. 查看你刚创建的配置

你也可以通过 API 获取：

```bash
curl http://localhost:8080/rainbow-bridge/api/v1/runtime/config \
  -H "x-environment: dev" \
  -H "x-pipeline: main"
```

```
dev         - 开发环境
test        - 测试环境
staging     - 预发布环境
prod        - 生产环境
prod-us     - 美国生产环境
prod-eu     - 欧洲生产环境
```

### 渠道命名建议

```
main        - 主渠道
hotfix      - 紧急修复渠道
feature-xxx - 特定功能开发渠道
experiment  - 实验性功能渠道
```

### 配置命名规范

使用有意义的名称，推荐格式：

```
模块_功能_属性
api_base_url          # API 基础地址
ui_theme_color        # UI 主题色
feature_new_checkout  # 新功能开关
```

## ❓ 常见问题

### Q: 环境和渠道的区别？

**A**: 
- **环境**用于隔离不同的部署场景（如开发、测试、生产）
- **渠道**用于在同一环境下管理不同的功能分支或版本线

例如：
- 环境：`prod`（生产环境）
- 渠道：`main`（正式发布）、`hotfix`（紧急修复）

### Q: 配置删除了能恢复吗？

**A**: 当前版本不支持回收站功能，删除后无法恢复。建议：
- 定期使用导出功能备份配置
- 删除前仔细确认
- 使用配置迁移功能复制而非修改

### Q: 如何批量修改配置？

**A**: 
1. 使用导出功能导出配置
2. 本地编辑 JSON 文件
3. 使用导入功能更新

或者使用配置迁移功能在不同环境间同步。
