# 环境管理

环境是虹桥计划中用于隔离不同部署场景的核心概念。本指南将帮助你创建和管理环境。

## 概念说明

### 什么是环境？

环境（Environment）用于隔离不同的部署场景，例如：

| 环境标识 | 说明 | 典型用途 |
|---------|------|---------|
| `dev` | 开发环境 | 本地开发调试 |
| `test` | 测试环境 | 功能测试、集成测试 |
| `staging` | 预发布环境 | 上线前验证 |
| `prod` | 生产环境 | 线上服务 |

### 环境与渠道的关系

- **环境**：粗粒度隔离，如开发/测试/生产
- **渠道**：细粒度隔离，如主渠道/热修复渠道/功能渠道

一个环境可以包含多个渠道，形成层级结构：

```
prod (生产环境)
├── main (主渠道)
├── hotfix (热修复渠道)
└── experiment (实验渠道)

dev (开发环境)
├── main (主渠道)
└── feature-xxx (功能开发渠道)
```

## 创建环境

### 通过 UI 创建

1. 访问管理界面：`http://localhost:8080/rainbow-bridge`
2. 点击左侧导航栏的 **环境管理**
3. 点击 **新建环境** 按钮
4. 填写环境信息：
   - **环境标识**：唯一标识符，如 `prod`
   - **环境名称**：显示名称，如 `生产环境`
   - **备注**：可选，描述信息
5. 点击 **确定** 保存

### 通过 API 创建

```bash
curl -X POST http://localhost:8080/rainbow-bridge/api/v1/environment/create \
  -H "Content-Type: application/json" \
  -d '{
    "environment_key": "prod",
    "environment_name": "生产环境",
    "remark": "线上生产环境"
  }'
```

## 管理环境

### 查看环境列表

**通过 UI**：在环境管理页面可以看到所有环境列表

**通过 API**：
```bash
curl http://localhost:8080/rainbow-bridge/api/v1/environment/list
```

### 编辑环境

1. 在环境列表中找到目标环境
2. 点击 **编辑** 按钮
3. 修改环境名称或备注
4. 点击 **确定** 保存

::: warning 注意
环境标识（environment_key）创建后不可修改，只能修改环境名称和备注。
:::

### 删除环境

1. 在环境列表中找到目标环境
2. 点击 **删除** 按钮
3. 确认删除操作

::: danger 警告
删除环境将同时删除该环境下的所有渠道和配置，此操作不可恢复！
:::

## 最佳实践

### 环境命名规范

推荐使用简洁、有意义的标识符：

```
# 推荐的命名方式
dev        - 开发环境
test       - 测试环境
staging    - 预发布环境
prod       - 生产环境
prod-us    - 美国生产环境（多地域部署）
prod-eu    - 欧洲生产环境
```

### 环境隔离策略

根据团队规模选择合适的隔离方式：

**小型团队**：
```
dev → prod
```

**中型团队**：
```
dev → test → staging → prod
```

**大型团队/多地域**：
```
dev → test → staging → prod-us
                      → prod-eu
                      → prod-ap
```

### 安全建议

1. **生产环境保护**：考虑为生产环境设置访问权限控制
2. **审计日志**：定期检查生产环境的配置变更
3. **备份策略**：定期导出生产环境配置

## 常见问题

### Q: 环境标识可以修改吗？

A: 不可以。环境标识是系统的唯一标识，创建后不可修改。如需更改，需要创建新环境并迁移配置。

### Q: 删除环境后数据能恢复吗？

A: 不能。删除环境会级联删除该环境下的所有渠道和配置，建议删除前先导出备份。

### Q: 环境数量有限制吗？

A: 系统本身没有限制，但建议根据实际需要创建，过多的环境会增加管理复杂度。
